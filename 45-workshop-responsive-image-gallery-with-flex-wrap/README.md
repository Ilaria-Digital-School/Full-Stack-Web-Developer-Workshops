# Workshop: responsive image gallery with flex-wrap

**Theoretical Recap** Flex-wrap is a core Flexbox property that controls whether flex items are forced into a single line or can wrap onto multiple lines. By default, flex-wrap is set to nowrap, meaning all items stay on one row regardless of their size — this is the most common source of overflow bugs for beginners. Setting flex-wrap: wrap allows items to flow onto new rows (or columns, depending on flex-direction) when they run out of space, making it the cornerstone of responsive Flexbox layouts. The mental model to retain: think of flex items as words in a paragraph. Without wrapping, all words stay on one line and overflow the container. With wrapping, they naturally break onto the next line when space runs out. Combined with flex-basis (which sets the ideal size of each item) and flex-grow/flex-shrink (which control how items expand or contract), you get a powerful and predictable responsive system without writing a single media query for the grid logic itself. Common mistakes to avoid: forgetting to set a meaningful flex-basis on items (defaulting to auto makes wrapping unpredictable), using fixed pixel widths instead of percentages or relative units (blocking responsiveness), and neglecting gap or margin to create breathing room between wrapped items.
