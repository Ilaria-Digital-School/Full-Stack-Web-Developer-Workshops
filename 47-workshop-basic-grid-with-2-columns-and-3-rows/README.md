# Workshop: basic grid with 2 columns and 3 rows

**Theoretical Recap** CSS Grid is a two-dimensional layout system that lets you control both columns and rows simultaneously — a fundamental shift from Flexbox, which operates along a single axis. The core mental model: you define a grid container with `display: grid`, then describe its structure using `grid-template-columns` and `grid-template-rows`. Child elements (grid items) are automatically placed into the resulting cells, or you can position them explicitly using `grid-column` and `grid-row` properties. Key definitions to keep in mind: a **track** is a single row or column; a **cell** is the intersection of one row track and one column track; a **grid area** is any rectangular region spanning one or more cells. The `fr` unit (fractional unit) is essential — it distributes available space proportionally. For example, `grid-template-columns: 1fr 1fr` creates two equal-width columns that share all available space. Common mistakes at this stage: forgetting that `grid-template-rows` is optional if you only need auto-sized rows; confusing grid line numbers (they start at 1, not 0); and applying grid properties to the items instead of the container. A minimal working example: ```css .container { display: grid; grid-template-columns: 1fr 1fr; grid-template-rows: auto auto auto; gap: 16px; } ``` This creates a 2-column, 3-row grid with uniform gutters. Each direct child fills one cell in document order.
