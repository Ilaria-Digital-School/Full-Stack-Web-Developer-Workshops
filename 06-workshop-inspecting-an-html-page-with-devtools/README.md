# Workshop: inspecting an HTML page with DevTools

**Theoretical Recap** Browser DevTools is a built-in suite of developer tools available in every modern browser (Chrome, Firefox, Edge). It allows you to inspect, debug, and analyze any web page in real time — without modifying the actual source files. Think of it as an X-ray machine for the web: you can see the HTML structure, the applied CSS styles, network requests, JavaScript errors, and much more. The most important panel for now is the **Elements** (or Inspector) panel. It displays the live DOM — the Document Object Model — which is the browser's structured interpretation of the HTML source. Key distinction: the DOM and the raw HTML source are not always identical. The browser can modify the DOM dynamically (via JavaScript or auto-corrections), so what you see in DevTools may differ slightly from what is in the .html file. A common beginner mistake is confusing "View Page Source" (the raw file) with the DevTools Elements panel (the live DOM). Always use DevTools when you want to understand what the browser is actually rendering. Mini example: if you right-click on a button on any webpage and select "Inspect", DevTools opens directly on that element in the DOM tree. You can then read its tag, attributes, classes, and even temporarily modify its text or style to test ideas — changes disappear on page reload. Mastering DevTools early is non-negotiable as a developer.
