# Workshop: Reversal of the order of elements with flex-direction

**Theoretical Recap** The `flex-direction` property is one of the most fundamental properties of Flexbox. It defines the main axis along which flex items are laid out inside their container. It accepts four values: `row` (default, left to right), `row-reverse` (right to left), `column` (top to bottom), and `column-reverse` (bottom to top). Understanding this property is critical because it affects how all other Flexbox alignment properties — `justify-content`, `align-items`, `align-self` — behave, since those properties always refer to the main axis and the cross axis, both of which are determined by `flex-direction`. A key mental model: think of `flex-direction` as setting the direction of a river current. All your flex items float along that current. When you reverse it (`row-reverse` or `column-reverse`), items visually reorder without changing the HTML source order. This is powerful for responsive design — you can stack navigation links vertically on mobile and display them horizontally on desktop, or reverse a card layout on smaller screens without touching your markup. Common mistakes to avoid: confusing visual order with DOM order (screen readers and tab navigation still follow the HTML source, so accessibility can break if you rely solely on `flex-direction` for meaningful reordering), and forgetting that `row-reverse` also changes where `justify-content: flex-start` begins (it starts from the right side). Also, do not mix up `flex-direction: column` with `writing-mode` — they are not the same thing.
