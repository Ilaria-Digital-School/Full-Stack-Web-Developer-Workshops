# Workshop: making a table accessible (scope, summary)

**Theoretical Recap** HTML tables are powerful for displaying structured data, but they become confusing for screen reader users when accessibility attributes are missing. Two key tools address this: the `scope` attribute and the `summary` attribute (or its modern equivalent). The `scope` attribute is applied to `` elements to explicitly declare whether a header applies to a column (`scope="col"`), a row (`scope="row"`), a group of columns (`scope="colgroup"`), or a group of rows (`scope="rowgroup"`). Without `scope`, screen readers may struggle to associate data cells with their correct headers, especially in complex tables. The `summary` attribute was used directly on `` in HTML4, but is now deprecated in HTML5. The modern approach is to provide a description using either a `` element (visible to all users) or an `aria-describedby` attribute pointing to a hidden or visible paragraph that explains the table's structure and purpose. Do not confuse `` (a visible title) with a structural summary (an explanation of how the table is organized). Common mistakes include: applying `scope` only to column headers and forgetting row headers; writing a `` that merely repeats the table title without explaining its structure; and omitting any description for complex multi-axis tables. A well-accessible table should be fully understandable when navigated linearly by a screen reader.
