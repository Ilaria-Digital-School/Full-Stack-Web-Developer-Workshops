# Draw a simple organization chart

THEORETICAL RECAP An organization chart (or flowchart) is a visual representation of a process, algorithm, or decision logic. It translates the sequential, conditional, and repetitive structures of an algorithm into standardized graphic symbols. Each symbol has a precise meaning: rectangles represent actions or instructions, diamonds represent decision points (yes/no conditions), rounded rectangles (or ovals) mark the start and end of the process, and arrows indicate the direction of execution flow. This visual tool is the bridge between abstract pseudo-code and actual code. A common mistake beginners make is confusing the direction of arrows or placing decision points without clearly labeling both outcomes (yes and no branches). Another frequent error is drawing a chart that has no clear entry or exit point. Think of an organization chart as a map of your algorithm: every road must lead somewhere, and no road should be a dead end. For example, a simple process like 'Check if a number is even or odd' would start with an oval (Start), flow into a rectangle (Read the number), then into a diamond (Is the number divisible by 2?), with two branches: one leading to a rectangle (Display 'Even') and the other to a rectangle (Display 'Odd'), both reconnecting to a final oval (End). Keep this mental model in mind throughout the workshop.
