# Workshop: applying title levels correctly

**Theoretical Recap** HTML heading tags — from h1 to h6 — define the hierarchical structure of a document's content. Think of them as an outline: h1 is the main title of the page (used once per page, ideally), h2 breaks the content into major sections, h3 subdivides those sections, and so on down to h6. This hierarchy is not just visual — it is semantic. Screen readers use it to let users navigate a page by headings, and search engines use it to understand content priority. A clear mental model: imagine a book. The h1 is the book title, h2 values are the chapter titles, h3 values are sub-chapter headings, and h4–h6 handle deeper nesting when truly necessary. The most common mistakes beginners make are: skipping levels (jumping from h2 to h4), using heading tags purely for visual sizing instead of meaning, placing multiple h1 tags on a single page without good reason, and forgetting headings altogether inside section or article elements. Remember: never choose a heading level based on how it looks — use CSS for styling. Choose it based on where it sits in the document's logical outline.
