# Transform a cooking recipe into a pseudo-code

**Theoretical Recap** An algorithm is a finite, ordered sequence of unambiguous instructions designed to solve a problem or accomplish a task. Pseudo-code is a way of describing an algorithm using plain, structured language — somewhere between natural language and a programming language — without worrying about any specific syntax. It allows you to think through logic before writing actual code. A strong mental model: think of pseudo-code as giving instructions to someone who is extremely literal. They will do exactly what you say, in exactly the order you say it, nothing more. If you skip a step, it will fail. If you assume they already know something, they will get stuck. Key concepts to keep in mind: sequential execution (instructions run in order, one by one), conditional logic (IF something is true, THEN do this, ELSE do that), and loops (WHILE a condition holds, repeat these steps / FOR each item, do this). Variables store values you need to reuse (e.g., temperature, timer, quantity). Common mistakes beginners make: skipping implicit human knowledge (e.g., 'you know when an onion is golden'), forgetting to initialize variables, writing steps that are too vague to be actionable, and ignoring edge cases (what if an ingredient is missing?).
