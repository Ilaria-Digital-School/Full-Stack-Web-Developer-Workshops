# Workshop: blog layout (sidebar + content)

THEORETICAL RECAP CSS Grid is a two-dimensional layout system that lets you control both rows and columns simultaneously, making it the ideal tool for page-level layouts like a classic blog with a sidebar and a main content area. Unlike Flexbox, which operates along a single axis, Grid lets you define explicit tracks and place elements precisely within named or numbered areas. The key mental model: think of your page as a spreadsheet — you define the columns (e.g., 250px for the sidebar, 1fr for the content), you define the rows (header, main, footer), and you assign each child element to the appropriate cell or area using grid-template-areas or grid-column / grid-row shorthand. Common mistakes to avoid: forgetting to set the grid container (display: grid on the parent, not the children), using fixed pixel widths everywhere instead of combining fixed and fluid units (e.g., 250px 1fr), and neglecting the responsive breakpoint where the sidebar should stack below the content on small screens. A quick mental check: if your layout breaks on mobile, you likely forgot a media query that redefines grid-template-columns to a single column. Mini example: .page { display: grid; grid-template-columns: 250px 1fr; grid-template-rows: auto 1fr auto; grid-template-areas: 'header header' 'sidebar content' 'footer footer'; gap: 1rem; } This snippet gives you a two-column layout where the header and footer span both columns, and the sidebar sits to the left of the main content.
