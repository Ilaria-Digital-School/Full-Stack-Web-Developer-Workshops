# Workshop: insert a profile photo with caption

THEORETICAL RECAP The img tag is a void element in HTML — it has no closing tag. It requires at minimum two attributes: src, which points to the image file, and alt, which provides a textual description of the image for screen readers and search engines. The alt attribute is not optional in professional practice: an empty alt (alt="") is acceptable only for purely decorative images, but for a profile photo it must describe the content meaningfully (e.g., alt="Portrait of Jane Doe, web developer"). To go beyond a bare img tag and add semantic meaning to an image with its caption, HTML5 introduced the figure and figcaption elements. The figure element wraps the image (and optionally other content), while figcaption provides a visible, programmatically associated caption. This pairing communicates to browsers, screen readers, and search engines that the caption belongs to the image — something a plain p tag below an img cannot guarantee. A common mistake is placing the figcaption outside the figure, or omitting it entirely and using only a paragraph. Another frequent error is using a relative path incorrectly: if your HTML file is in a subfolder, src="images/photo.jpg" resolves relative to that file's location, not the project root.
