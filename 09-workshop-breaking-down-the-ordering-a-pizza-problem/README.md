# Workshop: breaking down the “ordering a pizza” problem

**Theoretical Recap** Problem decomposition is one of the most fundamental skills a developer must internalize. It consists of taking a complex, vague, or large problem and systematically breaking it down into smaller, independent, and manageable sub-problems. Each sub-problem should be simple enough to be solved in isolation before being reassembled into a coherent whole. This is the foundation of algorithmic thinking and it applies to every line of code you will ever write. A common mistake beginners make is trying to solve everything at once, jumping straight into writing code without understanding what the problem actually requires. This leads to confusion, bugs, and wasted time. The correct mental model is: understand first, decompose second, then solve step by step. Think of it like building with LEGO bricks — you identify the pieces you need before you start assembling anything. A mini example: if the problem is "send an email", you can decompose it into: (1) open an email client, (2) click "compose", (3) fill in the recipient field, (4) write a subject, (5) write the body, (6) click "send". Each step is atomic and clear. Notice that even this simple action has an order, dependencies, and conditions (what if there is no internet connection?).
