# Workshop: fix an HTML/CSS display bug with DevTools

THEORETICAL RECAP Browser DevTools are a built-in suite of tools available in every modern browser (Chrome, Firefox, Edge) that let you inspect, debug, and modify a web page in real time — without touching your source files. Think of DevTools as an X-ray machine for your page: you can see the HTML structure, the CSS rules applied to each element, the computed styles after inheritance and specificity, and even live-edit values to test fixes instantly. The core mental model: the browser reads your HTML, builds a DOM (Document Object Model) tree, then applies CSS rules in a specific order governed by specificity and inheritance. When something looks wrong visually, the bug almost always lives at the intersection of the DOM and the applied styles. DevTools lets you pinpoint exactly which rule is responsible — and whether it is being overridden, missing, or misapplied. Common beginner mistakes include: editing the source file immediately without first diagnosing the problem in the browser, confusing an inherited property with an explicitly set one, overlooking the box model (margin vs. padding vs. border), and not realizing that a rule has been crossed out in DevTools because a more specific rule is overriding it.
