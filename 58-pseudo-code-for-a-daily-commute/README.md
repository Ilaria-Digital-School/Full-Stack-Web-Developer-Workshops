# Pseudo-code for a daily commute

THEORETICAL RECAP An algorithm is a finite, ordered sequence of instructions designed to solve a problem or accomplish a task. Pseudo-code is the human-readable representation of an algorithm: it uses plain language combined with programming-like structures (conditions, loops, sequences) to describe logic without being tied to any specific programming language. Think of pseudo-code as a blueprint — architects draw plans before construction begins; developers write pseudo-code before they write actual code. Three key structures underpin pseudo-code: SEQUENCE (instructions executed one after another, in order), CONDITION (IF... THEN... ELSE... blocks that handle decision points), and LOOP (WHILE or FOR blocks that repeat actions). A common beginner mistake is skipping steps that feel 'obvious' in real life but are critical for a machine — for example, forgetting to check whether you have enough money before boarding a bus. Another frequent error is mixing the WHAT with the HOW: pseudo-code describes logic, not implementation details. Mini-example: START SET alarm to 7:00 AM WHEN alarm rings, wake up IF it is raining THEN take umbrella ELSE leave umbrella at home Go to the bus stop END This example shows sequencing and a simple conditional.
