# Workshop: list of external resources (target and rel)

**Theoretical Recap** When building web pages, links that point outside your own site — external resources, documentation, tools, partner pages — require specific handling to ensure a good user experience and proper security. Two HTML attributes are central here: `target` and `rel`. The `target` attribute on an `` tag controls where the linked document opens. The most commonly used value is `_blank`, which opens the link in a new browser tab or window. Other values include `_self` (default, same tab), `_parent`, and `_top`, but in day-to-day frontend work, `_blank` is the one you will use most. The `rel` attribute describes the relationship between your page and the linked resource. For external links, the two critical values are `noopener` and `noreferrer`. When you use `target="_blank"` without `rel="noopener noreferrer"`, you expose your page to a security vulnerability called "tabnapping": the newly opened page can access and manipulate the original page via `window.opener`. Adding `rel="noopener noreferrer"` closes this vulnerability and also prevents the destination site from knowing where the traffic came from (referrer information).
