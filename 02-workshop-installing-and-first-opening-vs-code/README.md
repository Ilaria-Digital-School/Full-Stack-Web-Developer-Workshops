# Workshop: installing and first opening VS Code

**Theoretical Recap** A development environment is the set of tools a developer uses daily to write, test, and run code. Among these tools, the code editor holds a central place. Visual Studio Code (VS Code) is a free, open-source, and cross-platform editor developed by Microsoft. It has become the industry standard for web development due to its speed, flexibility, and rich ecosystem of extensions. Unlike a basic text editor (like Notepad), VS Code understands code: it highlights syntax, detects errors in real time, integrates a terminal, and connects to version control systems like Git. A common beginner mistake is confusing a code editor with an IDE (Integrated Development Environment). VS Code sits in between: it is a lightweight editor that can be extended to behave like a full IDE. Another frequent error is skipping the installation of essential extensions right from the start, which limits productivity early on. Think of VS Code as your workshop table: the quality of your setup directly impacts the quality and speed of your work. Getting it right from day one is a professional reflex every developer must build. **Workshop Brief** This workshop puts into practice what you have learned about development environments and their components. Your mission is to install Visual Studio Code on your machine, perform a first configuration, and verify that everything is working correctly.
