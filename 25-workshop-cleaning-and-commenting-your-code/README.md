# Workshop: cleaning and commenting your code

**Theoretical Recap** Clean code and meaningful comments are not optional refinements — they are professional standards. Clean HTML means your code is readable at a glance: consistent indentation (typically 2 or 4 spaces per level), no orphaned tags, no redundant or empty elements, and a logical nesting structure. Every child element must be visually indented relative to its parent. Comments in HTML use the syntax and serve to explain intent, mark sections, or flag areas requiring future work. They are invisible to the end user but invaluable to any developer reading the file, including your future self. A common mistake beginners make is over-commenting — narrating the obvious ('this is a paragraph') rather than explaining the why or the what ('navigation for mobile menu, toggled by JS'). Another frequent error is inconsistent indentation: mixing tabs and spaces, or forgetting to indent nested elements, which makes the structure impossible to read at scale. A messy file is a liability in a team environment and a red flag in a technical interview. Mental model: think of your HTML file as a legal document. Every section must be clearly delimited, every clause properly indented, and annotations must add value, not noise.
