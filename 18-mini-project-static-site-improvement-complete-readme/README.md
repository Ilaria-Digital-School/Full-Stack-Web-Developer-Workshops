# Mini-project: static site improvement + complete README

## Overview This mini-project is the capstone of the introductory chapter 'Getting started: development environment and developer mindset'. It brings together everything you have practiced so far: debugging HTML/CSS with DevTools, using the test-commit cycle, researching solutions on MDN and StackOverflow, and writing professional documentation in Markdown. You will take an existing (or provided) static website with intentional bugs and quality issues, fix it systematically, and document the entire project with a complete, professional README. --- ## Project Description You are given a starter static site (HTML + CSS) that contains a series of display bugs, structural issues, and missing best practices. Your job is to: 1. **Diagnose and fix all bugs** using a structured debugging approach and browser DevTools. 2. **Improve the code quality** by applying clean HTML/CSS conventions. 3. **Version your work properly** using Git with small, meaningful commits following the test-commit cycle. 4. **Write a complete README** in Markdown that documents the project clearly and professionally. This project simulates a real junior developer task: taking ownership of an existing codebase, fixing it, and handing it off with proper documentation. --- ## Skills Validated - Systematic debugging of HTML/CSS bugs using browser DevTools - Applying the step-by-step debugging method (reproduce → isolate → identify → fix → verify) - Using the test-commit cycle: small, incremental, verifiable commits - Researching solutions effectively on MDN and StackOverflow - Writing a professional README in Markdown - Using Git for version control throughout the project
