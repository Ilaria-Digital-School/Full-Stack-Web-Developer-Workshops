# mini-project: creating your semantic resume

## Project Overview You will build a fully semantic HTML resume (curriculum vitae) from scratch. This is a real-world document you can actually use — your own professional profile structured with clean, accessible, and meaningful HTML. No CSS, no JavaScript: pure HTML semantics, structure, and accessibility. This mini-project is the capstone of the 'HTML and Semantic Structuring' chapter. It consolidates everything you have practiced so far: ARIA landmarks, semantic tags, heading hierarchy, clean indentation, HTML comments, and file/folder best practices. --- ## Skills Validated - Structuring a real document using appropriate HTML5 semantic elements - Applying a correct and accessible heading hierarchy (h1–h6) - Integrating ARIA landmarks where relevant - Writing clean, well-indented, and commented HTML code - Organizing files and folders following professional naming conventions - Producing accessible, standards-compliant HTML that a browser and a screen reader can parse correctly --- ## Project Description Your resume must represent your actual or fictional professional profile and include the following sections: 1. **Header** — Your name (h1), current job title or target role, and contact information (email, LinkedIn, GitHub, phone) 2. **Professional Summary** — A short paragraph describing who you are and what you are aiming for 3. **Work Experience** — At least two positions, each with job title, company name, dates, and a list of responsibilities or achievements 4. **Education** — At least one diploma or training, with institution name, degree title, and year 5. **Skills** — A categorized list of technical and soft skills 6. **Optional sections** — Languages,...
