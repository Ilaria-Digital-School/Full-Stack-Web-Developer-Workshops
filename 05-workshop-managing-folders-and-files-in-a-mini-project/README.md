# Workshop: Managing folders and files in a mini-project

THEORETICAL RECAP Before diving into the workshop, let's consolidate what you've learned about the terminal and file system management. The file system is organized as a tree: a root directory contains folders, which themselves contain subfolders and files. Every developer project starts with a clean, logical folder structure — this is not optional, it is a professional reflex. Key commands to know: `pwd` (print working directory — where am I?), `ls` (list contents of the current directory), `cd folder_name` (move into a folder), `cd ..` (go up one level), `mkdir folder_name` (create a new directory), `touch file_name` (create a new empty file), `rm file_name` (delete a file), `rm -r folder_name` (delete a folder and its contents recursively). A common mistake beginners make is running commands in the wrong directory — always check where you are with `pwd` before executing anything. Another frequent error is confusing relative paths (`./subfolder`) and absolute paths (`/Users/yourname/projects/subfolder`). Keep in mind: a well-structured project folder reflects a well-structured developer's mind. For example, a basic web project might look like this: `my-project/` containing `index.html`, a `css/` subfolder with `style.css`, and a `js/` subfolder with `main.js`. This simple mental model will guide everything you build going forward. --- WORKSHOP BRIEF In this variation workshop, you will apply your terminal skills to build, from scratch, the folder and file structure of a realistic mini web project.
