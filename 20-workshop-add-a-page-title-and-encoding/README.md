# Workshop: add a page title and encoding

THEORETICAL RECAP Before diving into the workshop, let's consolidate the essentials. The section of an HTML document is not visible in the browser window, but it is critical: it contains metadata that browsers, search engines, and assistive technologies rely on to correctly interpret and display your page. Two elements are non-negotiable in any professional HTML document. First, tells the browser which character encoding to use. UTF-8 is the universal standard: it supports virtually every character in every language, including accented letters, special symbols, and emoji. Without it, characters like é, ü, or ñ may display as garbled text — a common and embarrassing mistake in production. Always place this tag as the very first child of , before anything else, so the browser applies the encoding from the start. Second, the element defines the text that appears in the browser tab, in bookmarks, and in search engine result pages. A good title is concise (50–60 characters), descriptive, and meaningful out of context. A bad title — or no title at all — harms both usability and SEO. Think of it as the first thing a user reads before even opening your page. Mental model: the is a control panel for the browser. The charset is the language decoder, and the title is the page's identity card. Get these two right on every single page you build.
