# Workshop: Fluid grid with repeat (auto-fit, minmax)

**Theoretical Recap** CSS Grid's `repeat()` function combined with `auto-fit` and `minmax()` is one of the most powerful tools for building truly fluid, responsive layouts without a single media query. Here is what you need to understand before starting. `minmax(min, max)` defines a size range for a grid track: the track will never be smaller than `min` and never larger than `max`. Using `1fr` as the maximum allows the track to grow and fill available space proportionally. `repeat(n, track-size)` simply repeats a track definition `n` times. The magic happens when you replace `n` with `auto-fit` or `auto-fill`. `auto-fit` collapses empty tracks and stretches existing items to fill the row. `auto-fill` keeps empty tracks, preserving column slots even if no item occupies them. For fluid card-based layouts, `auto-fit` is almost always what you want. A canonical pattern: `grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));` — this tells the browser: create as many columns as fit in the container, each at least 250px wide, and let them grow equally. No media query needed. Common mistakes to avoid: setting `min` in `minmax()` to a percentage (this can cause overflow), forgetting to set a meaningful `gap`, or confusing `auto-fit` with `auto-fill` and wondering why items do not expand. Always use a fixed or viewport-relative unit like `px` or `rem` as the minimum value. --- **Workshop Brief** You have just covered the fundamentals of CSS Grid — containers, tracks, rows, columns, named areas, and best practices for naming grid areas.
