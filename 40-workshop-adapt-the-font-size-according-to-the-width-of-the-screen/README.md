# Workshop: Adapt the font size according to the width of the screen

THEORETICAL RECAP Responsive typography is one of the most impactful aspects of responsive design. The core idea is simple: text that reads comfortably on a desktop can become too small or too large on a mobile screen if left unchanged. CSS gives us several tools to handle this. The most fundamental approach uses media queries — which you have already practiced — to define different font-size values at specific breakpoints. For example, a paragraph might be set to 14px on small screens and 18px on larger ones. This is explicit and predictable. Beyond fixed values, CSS3 introduced relative and fluid units: em and rem scale relative to a parent or root element, while viewport units (vw, vh) tie sizing directly to the screen dimensions. A font-size of 2vw means the text will always be 2% of the viewport width — it scales continuously without needing breakpoints. The clamp() function goes further by letting you define a minimum, a preferred fluid value, and a maximum, all in one declaration: clamp(14px, 2.5vw, 22px). Common mistakes to avoid: using only px for all font sizes (breaks accessibility scaling), abusing vw alone without clamping (text becomes unreadable at extreme sizes), and forgetting that line-height and spacing also need to adapt alongside font size.
