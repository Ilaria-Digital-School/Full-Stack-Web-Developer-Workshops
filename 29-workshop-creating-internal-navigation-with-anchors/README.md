# Workshop: creating internal navigation with anchors

THEORETICAL RECAP Internal navigation with anchors is one of the oldest and most practical features of HTML. An anchor is created using the tag, which you already know from the previous lesson on hypertext links. When used for internal navigation, the tag does not point to an external URL but to a specific location within the same page. This is done in two steps: first, you assign a unique id attribute to the target element (for example, ); second, you create a link pointing to that id using the hash symbol (for example, Go to Skills). The mental model is simple: the id is the destination, the href with # is the GPS coordinate that sends the user there. Common mistakes include forgetting the # in the href, using duplicate id values on the same page (ids must be unique), or placing ids on elements that are hard for the browser to scroll to. Another frequent error is confusing the anchor used for internal navigation with the anchor used for external links — they use the same tag, but the href value is fundamentally different. A quick example: if your page has a section , a link like See my experience placed at the top of the page will instantly scroll the user to that section. This pattern is widely used in single-page layouts, documentation pages, and long-form resumes — exactly the kind of document you have been building throughout this chapter.
