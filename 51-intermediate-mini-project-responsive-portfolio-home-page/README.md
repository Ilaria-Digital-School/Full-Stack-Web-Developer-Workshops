# Intermediate mini-project: responsive portfolio home page

## Project Overview You will design and build a fully responsive portfolio home page from scratch using HTML5 and CSS3, with a strong focus on CSS Grid and Flexbox. This mini-project is the culmination of your work on responsive design: grid vocabulary, fluid grids, alignment, placement, media queries, and layout splitting. This is not a theoretical exercise — you are building a real, deployable page that could serve as the foundation of your personal developer portfolio. Every layout decision must be intentional and justified by the techniques studied in this chapter. --- ## Project Description The portfolio home page must include the following sections: 1. **Header / Navigation bar** — Logo on the left, navigation links on the right. Must use Flexbox. Must be responsive (hamburger menu or stacked layout on mobile is acceptable). 2. **Hero section** — A full-width banner with your name, job title, and a call-to-action button. Layout built with Flexbox or Grid. 3. **Skills section** — A grid of skill cards (at least 6 skills). Must use CSS Grid with `repeat(auto-fit, minmax(...))` for fluid responsiveness. 4. **Projects section** — A grid of at least 4 project cards, each showing a project title, a short description, and a tag or category. Must use named Grid areas or explicit Grid placement. 5. **Sidebar + Main content section** — A section combining a sidebar (e.g., contact info, social links, availability status) and a main content area (e.g., a short bio or featured project).
