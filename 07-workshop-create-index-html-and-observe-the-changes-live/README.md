# Workshop: create index.html and observe the changes live

**Theoretical Recap** A web page is fundamentally a text file interpreted by a browser. The file named index.html holds a special status: by convention, web servers serve it automatically when no specific file is requested, making it the entry point of any web project. An HTML file is structured around a DOCTYPE declaration, an root element, a section (for metadata, title, and future stylesheets), and a section (for visible content). The browser reads this file top to bottom, parses the HTML tags, builds the DOM (Document Object Model), and renders the result visually. A common beginner mistake is confusing the file path shown in the browser address bar (a local file:// URL) with a real web server URL. At this stage, you are working locally: your browser reads the file directly from your disk. This is perfectly valid for learning and development. Another frequent error is forgetting to save the file before refreshing the browser — the browser always reads the last saved version on disk, not what is currently in your editor. Mental model: think of your workflow as a three-part loop — Edit (VS Code) → Save → Refresh (browser). Each iteration of this loop is a development cycle. As a developer, you will repeat this loop thousands of times. The faster and more natural it becomes, the more productive you will be. Example: create a file index.html, type a heading tag with your name inside it, save, open in browser, see the result. Change the text, save...
