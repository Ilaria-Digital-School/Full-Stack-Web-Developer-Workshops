# Workshop: basic alignment with Flexbox

**Theoretical Recap** Flexbox (Flexible Box Layout) is a CSS layout model designed to distribute space and align items inside a container along one axis — either horizontal (row) or vertical (column). The key mental model is this: you have a **flex container** (the parent) and **flex items** (the direct children). The container controls the overall flow; the items can optionally override their own alignment. The most essential properties on the container are: `display: flex` (activates the model), `flex-direction` (row or column), `justify-content` (alignment along the main axis: start, center, space-between, space-around, space-evenly), `align-items` (alignment along the cross axis: stretch, center, flex-start, flex-end, baseline), and `flex-wrap` (whether items wrap to the next line or not). On the item side, `align-self` allows a single item to override the container's `align-items` rule. Common mistakes to avoid: confusing the main axis and the cross axis (they swap when you change `flex-direction`), forgetting that `justify-content` has no effect when items fill the full main axis, and applying flex properties to grandchildren instead of direct children. A quick mental check: always ask yourself which axis you are working on before choosing a property. Mini example: to center a button both horizontally and vertically inside a card, set `display: flex`, `justify-content: center`, and `align-items: center` on the card — three lines, job done. --- **Workshop Brief** You have just studied the fundamental principles of Flexbox — the flex container, flex items, flow direction, and automatic margins.
