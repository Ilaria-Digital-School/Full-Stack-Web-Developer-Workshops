# Workshop: building a weekly schedule

THEORETICAL RECAP HTML tables are one of the most semantically rich structures in the language — when used correctly. A table is designed to represent tabular data: information that has a logical relationship between rows and columns. The key elements are: (the container), (a descriptive title for the table), (the header section), (one or more body sections for data rows), (a table row), (a header cell, which accepts the 'scope' attribute to indicate whether it applies to a column or a row), and (a standard data cell). The 'scope' attribute on is critical for accessibility: scope="col" indicates the header applies to all cells in its column, while scope="row" indicates it applies to all cells in its row. A common mistake is using tables for layout purposes (to position blocks of content visually). This is semantically incorrect and harmful to accessibility. Tables must only be used when the data genuinely has a two-dimensional relational structure. Another frequent error is forgetting or omitting and , which reduces both the semantic clarity and the screen reader compatibility of your table. Also, developers often forget to add 'scope' to their elements, leaving assistive technologies without context. Mini example: a simple two-column table listing days and tasks should have a like 'Weekly Task Overview', a with for 'Day' and 'Task', and a with rows containing cells for the actual data.
