# Workshop: go from 4 columns to 1 column depending on the screen size

**Theoretical Recap** CSS Grid is a two-dimensional layout system that lets you define rows and columns explicitly. When combined with media queries, it becomes a powerful tool for building truly responsive layouts. The core idea behind responsive Grid is simple: you define a layout for large screens, then override it progressively for smaller breakpoints using @media rules. The key properties you will rely on are grid-template-columns, which controls how many columns exist and how wide they are, and gap, which manages spacing between tracks. A common pattern is to start with grid-template-columns: repeat(4, 1fr) for desktop, then reduce the column count at each breakpoint: 3 columns for tablet landscape, 2 columns for tablet portrait, and finally 1 column for mobile. The 1fr unit means one fraction of the available space, ensuring columns share space equally and fluidly. A frequent mistake is forgetting that media queries do not add specificity — they only apply when the condition is met. If your base styles are too specific, breakpoints may not override them. Another common error is using fixed pixel widths inside a Grid that should be fluid, which breaks the layout at unexpected sizes. Mini example: .grid { display: grid; grid-template-columns: repeat(4, 1fr); gap: 1rem; } @media (max-width: 900px) { .grid { grid-template-columns: repeat(2, 1fr); } } @media (max-width: 480px) { .grid { grid-template-columns: 1fr; } } This pattern, combined with what you have already practiced in the blog layout and Grid media queries lessons, is exactly what this workshop asks you...
