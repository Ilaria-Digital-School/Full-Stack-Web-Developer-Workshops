# Final mini-project: fully semantic multi-page portfolio

## Project Overview This final mini-project is the capstone of the 'HTML and Semantic Structuring' chapter. You will design and build a **fully semantic, multi-page personal portfolio website** using only HTML (no CSS styling is required, though basic CSS is welcome). This project consolidates every concept covered in the chapter: document structure, semantic sectioning, inline tags, accessible tables, and an end-to-end semantic strategy. The portfolio must present you as a future Full Stack Web Developer and include at minimum the following pages: 1. **Home page** (`index.html`) — A short professional introduction. 2. **About page** (`about.html`) — Your background, skills, and career transition story. 3. **Projects page** (`projects.html`) — A showcase of at least two (real or fictional) projects, presented using appropriate semantic elements. 4. **Skills table page** (`skills.html`) — A structured, accessible table listing your technical skills by category, proficiency level, and learning status. 5. **Contact page** (`contact.html`) — Contact information and/or a simple HTML form. ## Skills Being Validated This project directly validates your ability to: - Structure an HTML document from scratch using correct, meaningful semantic elements. - Apply inline semantic tags (``, ``, ``, ``, ``, ``, etc.) in appropriate contexts. - Build accessible, well-structured HTML tables using ``, ``, ``, ``, ``, ``, and `scope` attributes. - Implement a coherent, end-to-end semantic structuring strategy across multiple pages. - Write clean, readable, and standards-compliant HTML code.
