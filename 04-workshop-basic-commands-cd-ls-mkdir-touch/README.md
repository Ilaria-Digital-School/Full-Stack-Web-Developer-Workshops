# Workshop: basic commands (cd, ls, mkdir, touch)

**Theoretical Recap** The terminal is a text-based interface that allows you to communicate directly with your operating system using commands. Unlike a graphical interface where you click on folders and files, the terminal requires you to type instructions precisely. This is how developers navigate their file system and organize projects every day. Four commands form the absolute foundation of terminal navigation and file management. `ls` (list) displays the contents of the current directory — think of it as opening a folder and seeing what is inside. `cd` (change directory) moves you from one directory to another — it is your way of walking through the file tree. `mkdir` (make directory) creates a new folder at the location of your choice. `touch` creates a new empty file instantly. A critical mental model: at any moment in the terminal, you are standing somewhere in the file system. This is called your current working directory. All commands you type are executed relative to where you currently are. The most common beginner mistake is running a command without knowing where you are, which leads to creating files or folders in unexpected locations. Always use `ls` and `pwd` (print working directory) to orient yourself before acting. Mini example: you open your terminal, type `pwd` and see `/Users/yourname`. You type `ls` to see existing folders. You type `mkdir my-project` to create a folder, then `cd my-project` to enter it, then `touch index.html` to create a file inside. You are now organized and ready to work. ---...
