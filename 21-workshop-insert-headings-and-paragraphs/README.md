# Workshop: insert headings and paragraphs

**Theoretical Recap** HTML headings and paragraphs are the backbone of any readable web page. Headings are defined with the tags h1 through h6, where h1 represents the most important title (usually the page's main topic) and h6 the least important. Each page should contain exactly one h1 tag — this is a fundamental SEO and accessibility best practice. Headings create a logical hierarchy, like a table of contents: h2 sections sit beneath h1, h3 beneath h2, and so on. Never skip heading levels (jumping from h1 directly to h4, for example) as this breaks the document outline and confuses assistive technologies like screen readers. Paragraphs are wrapped in p tags. Each p element creates a distinct block of text separated by vertical spacing in the browser. A common mistake is to use line breaks (br) to separate blocks of text instead of proper p tags — this is bad practice and should be avoided. Another frequent error is nesting block-level elements like p inside other p tags, which is invalid HTML. Mental model: think of your HTML document as a newspaper article. The h1 is the main headline, h2 tags are section headers, h3 tags are sub-section headers, and p tags are the body paragraphs. Structure always comes before styling.
