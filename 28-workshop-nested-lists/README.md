# Workshop: Nested lists

THEORETICAL RECAP A nested list is a list that contains one or more other lists inside one of its items. In HTML, this means placing a , , or element directly inside a element of a parent list. The browser will automatically indent the child list, visually reflecting the hierarchy of information. This structure is extremely common in real-world interfaces: navigation menus, multi-level table of contents, categorized skill sets, project breakdowns, and more. The mental model to keep in mind is simple: think in levels. Level 1 is the main list. Level 2 is a list nested inside a of level 1. Level 3 is a list nested inside a of level 2, and so on. Each level must be a complete, valid list — opened and closed properly — before the parent is closed.
