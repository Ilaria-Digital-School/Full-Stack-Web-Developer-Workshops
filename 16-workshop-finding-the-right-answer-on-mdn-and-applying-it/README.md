# Workshop: finding the right answer on MDN and applying it

**Theoretical Recap** MDN Web Docs (Mozilla Developer Network) is the reference documentation for web technologies: HTML, CSS, JavaScript, and web APIs. Unlike a tutorial or a blog post, MDN is a specification-level resource — it tells you exactly what a property does, what values it accepts, which browsers support it, and provides minimal but precise examples. The key mental model to adopt is this: MDN is a map, not a GPS. It won't hold your hand step by step, but it will tell you exactly what every road looks like. A common beginner mistake is to land on MDN, feel overwhelmed by the density of information, and immediately leave for a Stack Overflow answer or a YouTube tutorial. The problem with that reflex is that secondary sources can be outdated, context-specific, or simply wrong. Learning to read MDN directly is a foundational professional skill. Another frequent mistake is copying an example from MDN without understanding the parameters — for instance, using `addEventListener` without knowing what the `event` object actually contains. Mini example: you want to make text bold using CSS. You search MDN for `font-weight`, find the property page, read that it accepts keyword values (`bold`, `normal`) and numeric values (`100` to `900`), check the browser compatibility table, and apply it with confidence. That is the workflow.
