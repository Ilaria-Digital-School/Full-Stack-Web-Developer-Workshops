# Mini-project: versioned static site (env + Git)

## Overview This mini-project is the culminating exercise of the 'Getting Started' chapter. You have learned how to set up a development environment, write meaningful commits, manage commit history, work with branches, and create a .gitignore file. Now it's time to bring all of these skills together in a single, concrete deliverable: a versioned static website. You will build a simple static site (HTML/CSS) from scratch and manage its entire development lifecycle using Git — from the first commit to a clean, readable project history. ## Context As a developer, every project you work on — no matter how small — must be version-controlled. Recruiters and team leads routinely look at candidates' Git repositories to assess professionalism, attention to detail, and ability to collaborate. A clean Git history is not optional: it is a core professional standard. This project simulates real working conditions: you will initialize a repository, build incrementally, commit at meaningful checkpoints, use branches, and ensure that unwanted files are excluded via .gitignore. ## Skills Validated - Setting up and using a local development environment - Initializing and managing a Git repository - Writing clear, meaningful commit messages following best practices - Reading and interpreting commit history with git log - Creating and merging branches - Configuring a .gitignore file adapted to an HTML/CSS project - Publishing a repository to a remote platform (GitHub or GitLab)
