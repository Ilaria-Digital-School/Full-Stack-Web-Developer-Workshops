# Workshop: adaptive "About" section

THEORETICAL RECAP Flexbox is a one-dimensional layout model that controls how items are distributed along a single axis — either a row or a column. The core mental model is simple: a flex container governs the behavior of its direct children (flex items) using properties like flex-direction, justify-content, align-items, flex-wrap, and gap. The children themselves can use flex, align-self, and order to override or refine that behavior. A classic mistake beginners make is applying flex properties to the wrong element — for instance, trying to center content by setting justify-content on a flex item instead of its parent container. Another frequent error is forgetting to set flex-wrap: wrap when the layout needs to reflow on smaller screens, which causes items to overflow instead of stacking. Here is a minimal mental model to keep in mind: the parent controls distribution and alignment globally; the children control their own size and order locally. For example, a two-column layout where text sits next to an image can be achieved with display: flex on the container, and then switching to flex-direction: column inside a media query for mobile screens.
