# observation: inspecting a non-responsive site

THEORETICAL RECAP Before diving into the workshop, let's consolidate the core concepts you encountered in the previous lessons. A responsive website is one that adapts its layout and content fluidly to any screen size — whether it's a smartphone, tablet, or desktop monitor. The foundation of responsive design rests on three pillars: a flexible grid (using relative units like percentages instead of fixed pixels), flexible media (images and videos that scale), and media queries (CSS rules that apply styles only when certain viewport conditions are met). The viewport is the visible area of a web page on a given device. On mobile, without the correct meta viewport tag (), a browser will render the page at desktop width and then scale it down, making everything tiny and unreadable. Breakpoints are threshold values (e.g., 768px, 1024px) at which the layout shifts to better suit the screen size. Fluidity means that elements resize smoothly between breakpoints rather than jumping abruptly. A common mistake beginners make is confusing a site that looks acceptable on a resized desktop browser with one that is truly responsive. A truly responsive site is designed mobile-first and tested on real devices or accurate device emulators. Another frequent error is relying solely on fixed-width containers (e.g., width: 960px) with no fluid fallback, which instantly breaks the layout on smaller screens. Mini example: if a div has width: 960px, it will overflow on a 375px-wide phone screen. Replace it with max-width: 960px; width: 100%; and it immediately behaves more responsively....
