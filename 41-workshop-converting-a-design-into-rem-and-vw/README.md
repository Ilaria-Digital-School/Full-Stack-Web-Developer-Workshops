# Workshop: converting a design into rem and vw

**Theoretical Recap** Relative units are the backbone of responsive design. Unlike pixels (px), which are fixed and absolute, rem and vw scale dynamically based on context. The rem unit (root em) is always relative to the font-size defined on the element. By default, browsers set this to 16px, meaning 1rem = 16px. If you set html { font-size: 62.5%; }, then 1rem = 10px, which makes mental calculations easier (e.g., 24px = 2.4rem). The vw unit (viewport width) represents 1% of the browser window's width, making it ideal for fluid typography and layout elements that must scale continuously with screen size. A common mistake is mixing px and rem/vw inconsistently, which breaks the scaling logic of your layout. Another frequent error is using vw for font sizes without a minimum size fallback, which can make text unreadable on very small screens. Use clamp() to set a minimum, preferred, and maximum size: clamp(1rem, 2.5vw, 2rem) ensures text never gets too small or too large. The mental model to retain: rem controls typographic hierarchy and spacing relative to the root; vw controls fluidity relative to the viewport. Together, they allow you to build layouts that adapt gracefully without relying on breakpoints for every single adjustment. Mini example: font-size: clamp(1rem, 2vw, 1.5rem); — this sets a fluid font size that scales between 16px and 24px depending on viewport width. --- **Workshop Brief** In this workshop, you will convert a static, pixel-based HTML/CSS design into a fully fluid layout using rem and vw units....
